# 权限控制测试用例

## 1. 目标

本用例集用于验证不同角色的接口访问边界是否符合预期。

## 2. 自动化已覆盖

- `WORKER` 无法访问商品管理接口
- `WORKER` 无法访问报表接口
- `WORKER` 查询回收记录时只能看到自己的记录
- `KEEPER` 可以执行回收审核

对应自动化测试文件：

- `server/src/test/java/com/gongdi/materialpoints/PermissionIntegrationTests.java`

## 3. 手工测试用例

### 用例 1：工人登录后只看到工人侧功能

步骤：

1. 使用 `worker01 / 123456` 登录 H5
2. 查看首页功能按钮

预期结果：

- 可看到“我的回收记录”
- 可看到“我的积分”
- 可看到“积分兑换”
- 不显示“回收登记”

### 用例 2：库管登录后只显示登记相关功能

步骤：

1. 使用 `keeper / 123456` 登录 H5
2. 查看首页功能按钮

预期结果：

- 可看到“回收登记”
- 可看到“回收记录”
- 不显示“我的积分”和“积分兑换”

### 用例 3：工人访问后台管理接口被拒绝

步骤：

1. 使用工人账号获取 Token
2. 访问商品管理或报表接口

预期结果：

- 返回权限不足错误

### 用例 4：库管可审核回收记录

步骤：

1. 使用库管账号登录
2. 调用回收审核接口

预期结果：

- 可审核通过回收记录
