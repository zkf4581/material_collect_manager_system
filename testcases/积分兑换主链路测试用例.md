# 积分兑换主链路测试用例

## 1. 目标

本用例集用于验证“查看积分 -> 选择商品 -> 提交兑换申请 -> 后台审核 -> 扣减积分与库存”的主链路是否可用。

## 2. 自动化已覆盖

- 工人发起兑换申请成功
- 审核通过后扣减积分
- 审核通过后扣减商品库存
- 生成 `EXCHANGE_DEDUCT` 积分流水
- 积分不足时审核失败

对应自动化测试文件：

- `server/src/test/java/com/gongdi/materialpoints/ExchangeOrderIntegrationTests.java`

## 3. 手工测试用例

### 用例 1：H5 查看积分

步骤：

1. 使用 `worker01 / 123456` 登录 H5
2. 进入“我的积分”

预期结果：

- 显示当前积分余额
- 能看到积分流水列表

### 用例 2：H5 发起兑换

步骤：

1. 进入“积分兑换”
2. 选择任一商品
3. 点击“申请兑换”

预期结果：

- 提示申请成功
- 在“我的兑换记录”中可看到一条 `SUBMITTED` 状态记录

### 用例 3：后台审核通过

步骤：

1. 登录后台
2. 进入“兑换审核”
3. 对一条 `SUBMITTED` 记录点击“通过”

预期结果：

- 状态变为 `APPROVED`
- 对应工人积分减少
- 对应商品库存减少

### 用例 4：后台驳回

步骤：

1. 在“兑换审核”中选择一条待审核记录
2. 点击“驳回”

预期结果：

- 状态变为 `REJECTED`
- 不扣积分
- 不扣库存
