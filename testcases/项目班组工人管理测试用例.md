# 项目班组工人管理测试用例

## 1. 目标

本用例集用于验证“项目管理 + 班组管理 + 工人管理”是否可用。

## 2. 自动化已覆盖

- 新增项目成功
- 编辑项目成功
- 新增班组成功
- 编辑班组成功
- 新增工人成功
- 编辑工人成功

对应自动化测试文件：

- `server/src/test/java/com/gongdi/materialpoints/ProjectWorkerManageIntegrationTests.java`

## 3. 手工测试用例

### 用例 1：新增项目

步骤：

1. 登录后台
2. 进入“项目管理”
3. 填写项目名称、地点、状态
4. 点击保存

预期结果：

- 提示创建成功
- 列表中出现新项目

### 用例 2：新增班组

步骤：

1. 进入“班组管理”
2. 选择所属项目
3. 填写班组名称和状态
4. 点击保存

预期结果：

- 提示创建成功
- 列表中出现新班组

### 用例 3：新增工人

步骤：

1. 进入“工人管理”
2. 填写姓名、手机号、状态
3. 点击保存

预期结果：

- 提示创建成功
- 列表中出现新工人

### 用例 4：编辑基础数据

步骤：

1. 分别在项目、班组、工人列表点击“编辑”
2. 修改名称或状态
3. 点击保存

预期结果：

- 提示更新成功
- 列表展示最新值
