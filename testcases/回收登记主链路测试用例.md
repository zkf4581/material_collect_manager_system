# 回收登记主链路测试用例

## 1. 目标

本用例集用于验证“登录 -> 上传图片 -> 提交回收记录 -> 审核通过 -> 积分入账”的主链路是否可用。

## 2. 自动化已覆盖

- 登录成功并获取当前用户
- 创建回收记录成功
- 创建回收记录缺少照片时报错
- 审核回收记录后生成积分账户与积分流水

对应自动化测试文件：

- `server/src/test/java/com/gongdi/materialpoints/AuthControllerIntegrationTests.java`
- `server/src/test/java/com/gongdi/materialpoints/RecycleRecordIntegrationTests.java`

## 3. 手工测试用例

### 用例 1：H5 登录

前置条件：

- 服务端已启动
- 使用演示账号

步骤：

1. 打开 H5 登录页
2. 输入 `keeper / 123456`
3. 点击登录

预期结果：

- 跳转到首页
- 可看到当前账号与角色信息

### 用例 2：上传图片并提交回收记录

步骤：

1. 进入“回收登记”
2. 选择项目、班组、工人、材料
3. 填数量、单位、完好度
4. 上传至少一张图片
5. 提交

预期结果：

- 提示提交成功
- 返回预计积分
- 在“回收记录”中可看到新记录

### 用例 3：缺少图片时拦截

步骤：

1. 进入“回收登记”
2. 只填写表单，不上传图片
3. 提交

预期结果：

- 提示“请至少上传一张照片”
- 不产生新记录

### 用例 4：后台或接口审核通过

步骤：

1. 获取待审核记录 ID
2. 调用审核通过接口

预期结果：

- 状态从 `SUBMITTED` 变为 `APPROVED`
- 对应工人积分账户余额增加
- 生成一条 `RECYCLE_AWARD` 积分流水
