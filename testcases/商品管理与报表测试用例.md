# 商品管理与报表测试用例

## 1. 目标

本用例集用于验证“后台商品维护 + 基础报表展示”是否可用。

## 2. 自动化已覆盖

- 创建商品成功
- 更新商品成功
- 报表概览接口返回正确统计
- 排行榜接口返回正确排序
- 材料回收排行接口返回正确统计
- 商品兑换排行接口返回正确统计

对应自动化测试文件：

- `server/src/test/java/com/gongdi/materialpoints/RewardAndReportIntegrationTests.java`

## 3. 手工测试用例

### 用例 1：新增商品

步骤：

1. 登录后台
2. 进入“商品管理”
3. 填写商品名称、积分、库存、状态
4. 点击保存

预期结果：

- 提示创建成功
- 列表出现新商品

### 用例 2：编辑商品

步骤：

1. 在商品列表点击“编辑”
2. 修改商品名称或库存
3. 点击保存

预期结果：

- 提示更新成功
- 列表同步刷新显示最新值

### 用例 3：查看基础报表

步骤：

1. 登录后台
2. 进入“基础报表”

预期结果：

- 能看到回收记录总数
- 能看到已通过回收数
- 能看到累计发放积分
- 能看到兑换申请总数
- 能看到已扣兑换积分
- 能看到商品种类数

### 用例 4：查看积分排行

步骤：

1. 打开“基础报表”
2. 查看积分排行表格

预期结果：

- 排行按积分余额降序显示
- 能看到工人 ID、项目 ID、余额

### 用例 5：查看材料回收排行

步骤：

1. 打开“基础报表”
2. 查看“材料回收排行”表格

预期结果：

- 能看到材料名称
- 能看到累计回收数量和单位
- 能看到累计积分

### 用例 6：查看商品兑换排行

步骤：

1. 打开“基础报表”
2. 查看“商品兑换排行”表格

预期结果：

- 能看到商品名称
- 能看到兑换数量
- 能看到兑换积分
