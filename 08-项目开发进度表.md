# 项目开发进度表

## 1. 使用说明

本文件是项目执行过程中的唯一进度总表。每次完成一个模块、接口、页面、脚本或部署动作后，都要更新本文件，确保后续任何 Agent 或开发者都能在最短时间内知道项目进行到哪一步。

更新原则：

1. 先改状态，再提交代码
2. 完成后必须补充完成时间、负责人、验证结果
3. 进入联调前必须明确依赖是否已满足
4. 不允许只写“已完成”，必须写清楚完成了什么

## 2. 状态标签说明

- `TODO`：未开始
- `IN_PROGRESS`：开发中
- `BLOCKED`：受阻塞
- `DONE`：代码已完成
- `VERIFIED`：已验证通过
- `RELEASE_READY`：可进入部署或试运行
