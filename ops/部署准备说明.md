# 部署准备说明

## 1. 当前部署方式

当前项目已补齐基础部署文件，可按以下方式运行：

- 本地开发：`ops/docker/docker-compose.local.yml`
- 单机生产预案：`ops/docker/docker-compose.prod.yml`

## 2. 生产容器规划

- `mysql`：MySQL 8
- `server`：Spring Boot 后端
- `client-h5`：H5 前端静态站点
- `admin-web`：后台前端静态站点

默认端口：

- 后端：`8080`
- H5：`8081`
- 后台：`8082`

## 3. 启动方式

在项目根目录执行：

```bash
docker compose -f ops/docker/docker-compose.prod.yml up -d --build
```

## 4. 环境变量

当前生产编排支持以下环境变量：

- `MYSQL_ROOT_PASSWORD`
- `MYSQL_DATABASE`

如未显式设置，将使用默认值：

- `MYSQL_ROOT_PASSWORD=root`
- `MYSQL_DATABASE=material_points`

## 5. 上线前建议

1. 修改数据库密码
2. 为前端设置正式访问域名
3. 为后端开启 HTTPS 反向代理
4. 将上传目录挂载到持久化磁盘
5. 将默认演示账号改为正式账号体系
6. 配置日志采集与备份策略

## 6. 当前限制

- 当前 `compose.prod` 仍偏单机部署
- 前端通过独立端口提供访问
- 还没有统一反向代理聚合域名
- 还没有对象存储接入

后续如要上阿里云或腾讯云，建议下一步补：

1. 云数据库
2. 对象存储
3. Nginx/Ingress 统一入口
4. 备份与监控
