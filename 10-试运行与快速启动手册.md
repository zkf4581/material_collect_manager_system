# 试运行与快速启动手册

## 1. 目标

本手册用于快速启动当前项目的本地试运行环境，确保后续可以稳定、重复地拉起系统，并直接通过浏览器访问。

适用场景：

1. 本地演示
2. 内部试运行
3. 启动后快速检查 H5 与后台是否可访问

## 2. 一键启动方式

### 2.1 推荐：Demo 模式

默认推荐使用 `demo` 模式，不依赖 Docker 数据库，适合我后续快速重启项目，也适合你现在直接打开浏览器试用。

在项目根目录执行：

```bash
bash ops/scripts/run-demo.sh
```

如果默认端口被占用，可指定端口：

```bash
BACKEND_PORT=18080 H5_PORT=15173 ADMIN_PORT=15174 bash ops/scripts/run-demo.sh
```

### 2.2 可选：Full 模式

如果要使用本地 MySQL + Redis 容器，可执行：

```bash
RUN_MODE=full bash ops/scripts/run-demo.sh
```

## 3. 一键停止方式

在项目根目录执行：

```bash
bash ops/scripts/run-demo.sh
```

脚本会自动完成：

1. 启动本地 MySQL 与 Redis
2. 启动 Spring Boot 后端
3. 启动 H5 前端
4. 启动后台前端
5. 生成运行日志与 PID 文件

在项目根目录执行：

```bash
bash ops/scripts/stop-demo.sh
```

## 4. 查看运行状态

在项目根目录执行：

```bash
bash ops/scripts/status-demo.sh
```

## 5. 默认访问地址

- H5：`http://localhost:5173`
- 后台：`http://localhost:5174`
- 后端：`http://localhost:8080`
- 健康检查：`http://localhost:8080/api/health`

## 7. 建议试运行顺序

### 7.1 工地材料回收

1. 使用 `keeper / 123456` 登录 H5
2. 进入“回收登记”
3. 选择项目、班组、工人、材料
4. 填数量、单位、完好度
5. 上传图片并提交

### 7.2 审核与发分

1. 使用 `admin / 123456` 或 `keeper / 123456` 登录后台
2. 进入“回收审核”
3. 找到提交记录并点击“审核通过”

### 7.3 积分兑换

1. 使用 `worker01 / 123456` 登录 H5
2. 查看“我的积分”
3. 进入“积分兑换”
4. 发起兑换申请

### 7.4 兑换审核与发放

1. 使用 `admin / 123456` 或 `keeper / 123456` 登录后台
2. 进入“兑换审核”
3. 先点击“通过”
4. 再点击“确认发放”

## 8. 日志文件位置

运行脚本会在根目录生成：

```text
.runtime/
├── backend.log
├── h5.log
├── admin.log
├── backend.pid
├── h5.pid
└── admin.pid
```

## 9. 常见问题

### 9.1 端口占用

如果 `5173`、`5174`、`8080`、`3306` 被占用，请先关闭已有进程，或调整配置。

### 9.2 前端打不开

先检查：

```bash
bash ops/scripts/status-demo.sh
```

再查看：

```bash
tail -n 50 .runtime/h5.log
tail -n 50 .runtime/admin.log
```

### 9.3 后端打不开

查看：

```bash
tail -n 100 .runtime/backend.log
```

### 9.4 Full 模式数据库未启动

执行：

```bash
docker compose -f ops/docker/docker-compose.local.yml ps
```

### 9.5 Demo 模式后端未启动

查看：

```bash
tail -n 100 .runtime/backend.log
```

如果需要重新拉起：

```bash
bash ops/scripts/stop-demo.sh
bash ops/scripts/run-demo.sh
```
