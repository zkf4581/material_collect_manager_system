# 总体技术架构

## 1. 总体架构选型

### 1.1 技术栈

- 手机端 H5：`Vue 3` + `Vite` + `Vant` + `Pinia` + `Vue Router`
- 管理后台：`Vue 3` + `Vite` + `Element Plus` + `Pinia` + `Vue Router`
- 服务端：`Spring Boot 3.x` + `Spring Web` + `Spring Security` + `MyBatis Plus` 或 `JPA`
- 数据库：`MySQL 8.x`
- 缓存：`Redis 7.x`
- 文件存储：本地磁盘，后续支持 `OSS / COS / MinIO`
- API 文档：`OpenAPI / Swagger`

### 1.2 选型理由

- `Vue 3` 适合同时支撑 H5 与后台两套前端
- `Vant` 更适合移动端表单录入
- `Element Plus` 适合后台表格与审批场景
- `Spring Boot` 更适合企业内部管理系统，稳定且扩展性好
- `MySQL` 足以满足第一版业务需求

## 6. 图片上传方案

### 6.1 第一阶段

- 前端通过 `multipart/form-data` 上传图片
- 服务端校验文件类型、大小、扩展名
- 文件存储在本地目录，如 `server/uploads/materials/`
- 数据库存储文件 URL、原始文件名、大小、上传人

### 6.2 第二阶段

- 抽象文件存储接口
- 切换到 `OSS / COS / MinIO`
- 业务代码只依赖统一的文件服务，不直接耦合本地磁盘

## 7. 本地开发架构

```text
浏览器(H5/后台)
   ↓
Vite 本地开发服务
   ↓
Spring Boot 本地服务
   ↓
MySQL / Redis / uploads
```

本地阶段建议：

- H5：`http://localhost:5173`
- 后台：`http://localhost:5174`
- 后端：`http://localhost:8080`
- 上传访问前缀：`http://localhost:8080/uploads/...`

## 8. 后续上云架构

### 8.1 初期上云

- 1 台云服务器
- `Nginx` 反向代理前后端
- `Spring Boot` 进程运行
- `MySQL` 可先与应用部署在同一台机器
- 上传文件先使用服务器磁盘

### 8.2 进阶上云

- 应用服务器与数据库分离
- 文件切对象存储
- Redis 独立部署
- 增加备份与监控

## 9. 核心设计原则

- 前后端分离
- H5 与后台独立工程，避免互相污染
- 服务端按领域拆分模块，避免单个大控制器
- 图片存储能力抽象，便于替换
- 积分流水单独建模，保证可追溯
