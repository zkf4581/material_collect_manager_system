# 项目启动准备

## 1. 启动目标

本阶段目标不是一次性做完整系统，而是把项目的基础约束、开发环境、协作边界和交付顺序先定清楚，确保后续多人并行开发时不混乱、不返工。

## 2. 开发前必须确认的业务项

### 2.1 组织维度

- 是否按“公司 -> 项目 -> 班组 -> 工人”四级组织管理
- 工人账号是系统创建，还是手机号注册
- 积分是否允许跨项目累计
- 兑换商品是项目级管理，还是公司级统一管理

### 2.2 材料维度

- 材料是否统一维护标准字典
- 材料名称是否允许自由录入
- 单位是否限定为标准单位，如 `kg`、`根`、`张`、`米`、`袋`
- 完好度是否参与积分计算

### 2.3 流程维度

- 回收登记后是否需要审核才能入积分
- 谁可以修改或作废回收记录
- 兑换申请是否需要两级审核
- 是否要求每条回收记录必须上传照片

## 3. 本地开发环境准备

### 3.1 开发工具

- `Git`
- `Node.js 20 LTS`
- `pnpm 9.x`
- `JDK 17`
- `Maven 3.9+`
- `MySQL 8.x`
- `Redis 7.x`
- `IDEA` 或 `VS Code`
- `Apifox` 或 `Postman`

### 3.2 前端环境

- 安装 `Node.js`
- 统一使用 `pnpm`
- 约定代码格式化工具：`ESLint` + `Prettier`
- 约定提交校验：`lint-staged` + `husky`

### 3.3 后端环境

- 使用 `JDK 17`
- 使用 `Spring Boot 3.x`
- 使用 `Maven Wrapper`
- 统一接入 `Checkstyle` 或 `Spotless`

### 3.4 数据库环境

- 本地启动 `MySQL 8`
- 建立独立数据库：`material_points`
- 编写初始化 SQL 与测试数据脚本
- 约定所有结构变更必须通过版本化脚本管理

## 4. 本地开发阶段建议

### 4.1 运行方式

- 前端本地 dev server
- 后端本地启动 Spring Boot
- MySQL 与 Redis 本地安装或 Docker 启动
- 上传文件先存本地目录，如 `./uploads`

### 4.2 环境划分

- `local`：每位开发者本机环境
- `dev`：团队共享开发环境
- `test`：集成测试环境
- `prod`：正式环境

### 4.3 配置管理

- 所有敏感配置使用环境变量
- 前后端接口地址使用环境配置文件区分
- 上传目录、JWT 密钥、数据库地址不得写死在代码里

## 5. 初期代码规范

### 5.1 通用规范

- 命名统一使用英文技术命名，业务文案使用中文
- 关键业务逻辑补充中文注释
- 删除无用代码，不保留长期废弃兼容逻辑
- 提交前必须通过格式化、静态检查、基础测试

### 5.2 Git 规范

- 主干建议使用 `main`
- 功能分支建议使用 `feature/xxx`
- 修复分支建议使用 `fix/xxx`
- 每个功能一个独立分支，避免多人混改

### 5.3 接口规范

- 统一返回结构
- 统一错误码
- 所有写操作记录操作人和时间
- 上传接口和业务接口分离

## 6. 项目初期最小交付

第一阶段只要求打通以下闭环：

1. 管理员维护材料与积分规则
2. 现场通过 H5 提交回收记录并上传照片
3. 系统自动计算积分
4. 工人查看积分与回收记录
5. 工人提交兑换申请
6. 后台审核兑换并扣减积分

## 7. 当前启动建议

建议先完成以下准备动作：

1. 建立代码仓库目录
2. 建立数据库初始化脚本
3. 建立前后端基础脚手架
4. 明确接口文档维护工具
5. 确认第一版所需字段字典
