# 任务标签与交接规范

## 1. 目标

通过固定标签体系、完成定义和交接模板，让不同 Agent 在多轮协作中始终能看懂当前项目状态，避免“代码做了一半但没人知道做到哪”的情况。

## 2. 标签设计原则

- 标签必须稳定，不随人变化
- 标签要能反映模块归属
- 标签要能区分阶段和状态
- 标签要能直接映射到目录、接口或数据表

## 3. 推荐标签体系

## 3.1 文档类

- `DOC-INIT`：项目初期文档
- `DOC-API`：接口契约与状态机
- `PLAN-TRACK`：进度跟踪与交接规范

## 3.2 前端类

- `H5-BOOT`：H5 工程骨架
- `H5-RECYCLE`：H5 回收登记
- `H5-POINT`：H5 积分查询
- `H5-REDEEM`：H5 兑换申请
- `ADM-BOOT`：后台工程骨架
- `ADM-BASE-DATA`：后台基础数据管理
- `PROJECT-MGMT`：项目管理
- `TEAM-MGMT`：班组管理
- `WORKER-MGMT`：工人管理
- `ADM-RECYCLE`：后台回收审核
- `ADM-REDEEM`：后台兑换审核
- `ADM-REPORT`：后台报表

## 3.3 服务端类

- `SRV-BOOT`：服务端骨架
- `AUTH-CORE`：登录与权限
- `FILE-UPLOAD`：图片上传与存储
- `SRV-RECYCLE`：回收记录服务
- `SRV-POINT`：积分账户与流水
- `SRV-REDEEM`：兑换服务
- `SRV-REPORT`：报表服务

## 3.4 数据库类

- `DB-INIT`：建库与初始化
- `DB-MIGRATION`：结构迁移
- `DB-SEED`：种子数据

## 3.5 质量与运维类

- `QA-INTEG`：联调测试
- `QA-REGRESSION`：回归测试
- `OPS-LOCAL`：本地环境
- `OPS-CLOUD`：上云部署

## 4. 状态标签

- `TODO`
- `IN_PROGRESS`
- `BLOCKED`
- `DONE`
- `VERIFIED`
- `RELEASE_READY`

状态含义约束：

- `DONE` 只表示代码完成，不代表验证通过
- `VERIFIED` 表示已经按定义完成验证
- `BLOCKED` 必须写明阻塞原因和依赖项

## 5. 完成定义

## 5.1 模块完成的最小条件

一个模块要从 `IN_PROGRESS` 进入 `DONE`，必须同时满足：

1. 代码已提交到对应分支
2. 相关文档已更新
3. 自测已完成
4. 已写明影响范围

## 5.2 模块进入 VERIFIED 的条件

1. 关键功能已按用例走通
2. 接口结构符合约定
3. 没有明显阻塞下游模块的问题
4. 进度表已更新

## 6. 交接模板

每次 Agent 完成一项任务后，必须按下面格式交接：

```text
【交接标签】
H5-RECYCLE

【当前状态】
DONE

【完成内容】
1. 完成回收登记页面
2. 完成拍照上传接入
3. 完成表单校验与提交

【改动范围】
1. client-h5/src/views/recycle/**
2. client-h5/src/components/upload/**
3. client-h5/src/api/recycle.ts

【依赖关系】
依赖 FILE-UPLOAD、BASE-DATA

【验证结果】
1. 本地页面可打开
2. 表单校验通过
3. 提交接口已对通 mock / 真接口

【待下游继续】
1. SRV-RECYCLE 接入真实审核逻辑
2. QA-INTEG 做端到端联调
```

## 7. 阻塞模板

如果模块被阻塞，必须这样记录：

```text
【交接标签】
SRV-POINT

【当前状态】
BLOCKED

【阻塞原因】
point_ledger 表结构未冻结，balance_after 字段含义待确认

【阻塞依赖】
DB-INIT

【建议处理人】
Agent D / Agent G

【恢复条件】
表结构冻结并更新接口契约文档
```

## 8. 推荐执行规则

1. 每完成一个标签任务，就更新 [08-项目开发进度表.md](/Users/zhangxiaolong/工地材料换积分/08-项目开发进度表.md)
2. 每个 PR 或每轮开发都引用对应标签
3. 一个 Agent 同时只持有少量标签，避免上下文过大
4. 大标签如果继续膨胀，要拆成子标签

## 9. 推荐命名规则

推荐格式：

```text
类型-模块
```

示例：

- `H5-RECYCLE`
- `SRV-POINT`
- `DB-INIT`
- `OPS-LOCAL`

如果需要更细粒度，可扩展为：

```text
类型-模块-子任务
```

示例：

- `SRV-RECYCLE-API`
- `SRV-RECYCLE-REVIEW`
- `H5-RECYCLE-FORM`

## 10. 结论

后续这个项目不应该依赖“谁还记得做到哪了”，而应该依赖：

1. 固定标签
2. 统一进度表
3. 标准交接模板
4. 明确的完成定义

这样哪怕隔几天再回来，或者切换不同 Agent，也能快速恢复上下文。
